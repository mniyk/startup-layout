# Startup Layout

Windows 起動時に複数のアプリを「指定した位置・サイズ」で開くデスクトップアプリ。

毎朝のウィンドウ配置作業を自動化するシンプルなツールです。

## 機能

- **プリセット管理** — アプリパス + ウィンドウサイズ + 位置を登録・編集・削除
- **一括起動** — 有効なプリセットを全て起動し、指定位置に配置
- **個別起動** — プリセットごとに個別起動
- **既存ウィンドウから取得** — 現在アクティブなウィンドウの情報を自動取得してプリセット作成
- **マルチモニタ対応** — 配置先のモニタを指定可能
- **永続化** — `%APPDATA%\startup-layout\presets.json` に自動保存

## 必要環境

- Windows 10 / 11
- Rust (stable) — [rustup.rs](https://rustup.rs) からインストール
- WebView2 ランタイム (Windows 11 は標準搭載)

## ビルド & 実行

```powershell
# クローン
git clone https://github.com/mniyk/startup-layout
cd startup-layout

# 実行 (初回は依存関係のコンパイルで数分かかります)
cargo run

# リリースビルド
cargo build --release
# 実行ファイル: target\release\startup-layout.exe
```

## 使い方

### プリセットの追加

1. **「+ 新規プリセット」** ボタンをクリック
2. 表示名・実行ファイルパス・位置・サイズを入力して「保存」

または

1. 対象アプリを起動してウィンドウを配置
2. **「既存ウィンドウから取得」** ボタンをクリック
3. 3 秒のカウントダウン中に対象ウィンドウをアクティブにする
4. 自動入力されたプリセットを確認して「保存」

### 起動

- **「▶ 全部起動」** — チェックが入った全プリセットを一括起動
- カード右の **「▶」** ボタン — 個別起動

### モニタ指定

編集ダイアログの「モニタ」ドロップダウンで指定したモニタを選択すると、
X・Y 座標をそのモニタの左上を原点として解釈します。

「座標通り」を選択した場合は、仮想デスクトップ全体の絶対座標で配置します。

## 注意事項

- **Electron アプリ (Slack, VSCode 等)** は起動に時間がかかるため、「遅延 (ms)」を 2000〜3000 ms に設定推奨
- 管理者権限が必要なアプリへの対応は未実装
- 一部のアプリはウィンドウ検出に失敗する場合があります（自前でウィンドウ管理をするアプリなど）

## データ保存先

```
%APPDATA%\startup-layout\presets.json
```
